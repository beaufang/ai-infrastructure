# 通用设计模式目录

本目录用于辅助识别代码中的设计模式。当你在分析代码时遇到某种结构，可以对照此目录确认模式名称和标准描述。

## 目录

- [架构模式](#架构模式)
- [数据流模式](#数据流模式)
- [创建型模式](#创建型模式)
- [结构型模式](#结构型模式)
- [行为型模式](#行为型模式)
- [并发模式](#并发模式)

## 架构模式

| 模式 | 核心思想 | 识别信号 |
|------|---------|---------|
| 分层架构 | 按职责分层，层间单向依赖 | controller → service → repository 目录结构 |
| 六边形架构 | 核心业务独立，通过端口与外部交互 | domain/core 目录 + adapters/ports 目录 |
| 微服务 | 独立部署的服务，通过 API 通信 | 多个独立部署单元 + API 网关 |
| 事件驱动 | 组件通过事件异步通信 | EventBus、MessageQueue、PubSub 组件 |
| 管道/过滤器 | 数据经过一系列处理阶段 | Pipeline、Stage、Filter、Middleware 类 |
| 插件架构 | 核心系统通过插件扩展功能 | Plugin、Extension、Addon 注册机制 |
| CQRS | 读写操作分离 | Command、Query 分开的处理器 |
| 事件溯源 | 状态通过事件序列重建 | EventStore、Event 类 + replay 机制 |

## 数据流模式

| 模式 | 核心思想 | 识别信号 |
|------|---------|---------|
| 单向数据流 | 数据从顶层向下流动 | Props/参数向下传递，事件向上冒泡 |
| 观察者/发布订阅 | 状态变更自动通知订阅者 | Observer、Subscriber、Listener 注册 |
| 响应式流 | 数据作为异步流处理 | Observable、Stream、Flow 操作符链 |
| 仓库模式 | 统一的数据访问抽象层 | Repository 接口 + 具体实现 |
| 数据映射器 | 对象与存储之间的双向转换 | Mapper、Converter、Serializer 类 |
| 工作单元 | 跟踪变更并统一提交 | UnitOfWork、Transaction、Session 对象 |

## 创建型模式

| 模式 | 核心思想 | 识别信号 |
|------|---------|---------|
| 工厂方法 | 由子类决定创建哪个对象 | Creator 类 + create/build 方法 |
| 抽象工厂 | 创建一组相关对象 | Factory 接口 + 多个产品创建方法 |
| 构建器 | 分步构建复杂对象 | Builder 类 + 链式调用方法 |
| 原型 | 克隆已有对象创建新实例 | clone/copy 方法 |
| 单例 | 全局唯一实例 | getInstance、shared、default 静态属性 |

## 结构型模式

| 模式 | 核心思想 | 识别信号 |
|------|---------|---------|
| 适配器 | 兼容不同接口 | Adapter 类，包装不兼容的接口 |
| 桥接 | 分离抽象与实现 | 抽象类引用实现接口 |
| 组合 | 树形结构统一处理 | Component 基类 + Leaf/Composite 子类 |
| 装饰器 | 动态添加行为 | Wrapper 类实现相同接口并增强 |
| 门面 | 简化复杂子系统 | Facade 类提供高层接口 |
| 代理 | 控制对象访问 | Proxy 类与目标实现相同接口 |

## 行为型模式

| 模式 | 核心思想 | 识别信号 |
|------|---------|---------|
| 责任链 | 请求沿链传递直到被处理 | Handler 链 + next/setNext 方法 |
| 命令 | 将操作封装为对象 | Command 对象 + execute/undo 方法 |
| 迭代器 | 顺序访问集合元素 | Iterator 对象 + next/hasNext 方法 |
| 中介者 | 集中管理组件间通信 | Mediator 类协调多个组件 |
| 备忘录 | 保存和恢复对象状态 | Memento 对象 + save/restore 方法 |
| 策略 | 可互换的算法族 | Strategy 接口 + 多个实现 |
| 模板方法 | 父类定义算法骨架 | 抽象方法由子类实现具体步骤 |
| 访问者 | 在不修改类的情况下添加操作 | Visitor 接口 + accept 方法 |

## 并发模式

| 模式 | 核心思想 | 识别信号 |
|------|---------|---------|
| 活跃对象 | 将方法调用与执行分离 | 命令队列 + 工作线程 |
| 锁/互斥 | 保护共享资源 | Lock、Mutex、Semaphore |
| 读写锁 | 允许并发读、独占写 | RWLock、SharedLock |
| 线程池 | 复用线程避免频繁创建 | ThreadPool、WorkerPool |
| Future/Promise | 异步操作的占位结果 | Future、Promise、Task、CompletableFuture |
| 生产者-消费者 | 解耦数据生产和消费 | Queue + Producer/Worker |
| 熔断器 | 防止级联故障 | CircuitBreaker 状态机 |

## 使用说明

此目录用于**辅助识别**，而非替代分析。当你发现代码中存在某种模式时：
1. 对照此目录确认模式名称
2. 使用标准名称描述该模式
3. 记录该项目对该模式的**独特变体或适配**——这些变体往往比标准模式更有价值
